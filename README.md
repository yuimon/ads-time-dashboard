# 曜日・時間帯別 広告パフォーマンス ダッシュボード

Google Ads の「曜日 × 時間帯」レポート（Excel/CSV）を、ブラウザでグラフとして見られるようにするツールです。
インターネット接続やサーバーは不要で、**`index.html` をダブルクリックするだけ**で開けます。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | ページ本体。これをブラウザで開く |
| `style.css` | 見た目（ライト/ダークモード自動対応） |
| `app.js` | グラフの描画ロジック |
| `data.js` | グラフに表示するデータ（Excelから変換したもの） |
| `make_data.py` | Excel/CSVから `data.js` を作り直すスクリプト |

## 画面でできること

- **KPIサマリー** — 期間合計の表示回数・クリック数・費用・CV・CTR・CPC・CVR・CPA
- **指標切り替え** — 上部のボタンで8指標を切り替えると、ヒートマップと棒グラフが連動して変わる
- **曜日×時間帯ヒートマップ** — 色が濃いほど値が大きい。セルにカーソルを合わせると全指標を表示
- **1日あたり平均** — 集計期間内で曜日ごとの日数が違う場合の補正（例: 水曜だけ8日間ある等）
- **主な気づき** — ピーク時間帯・最良/最悪CPAなどをデータから自動生成
- **全データ表** — ページ下部の折りたたみから168行すべてを確認可能

URLパラメータで初期表示も指定できます（例: `index.html?m=cpa&norm=1`）。
`m` は `imp / clicks / cost / conv / ctr / cpc / cvr / cpa` のいずれか、`norm=1` で1日あたり平均on。

## 別のデータで使う手順

### 1. Google Ads からレポートを出力する

レポートの条件:

- 行が **時間帯（0〜23）× 曜日** になっていること
  （Google Ads の分割項目で「曜日」と「時間帯」を指定したキャンペーンレポート）
- 列に **時間帯 / 曜日 / 表示回数 / クリック数 / 費用 / コンバージョン** が含まれること
- 「コンバージョン」列より右にある列は、CVアクション内訳として自動で取り込まれます
- 形式は `.xlsx` でも `.csv` でもOK

### 2. data.js を作り直す

コマンドプロンプト（またはターミナル）でこのフォルダに移動して:

```
python make_data.py "レポートファイルのパス.xlsx"
```

例（ひとつ上のフォルダにあるファイルを使う場合）:

```
python make_data.py "..\20260520-0709.xlsx"
```

成功すると `data.js` が上書きされ、期間・行数・合計値が表示されます。
シートが複数あるxlsxでも「時間帯」「曜日」列を持つシートを自動で探します。
特定のシートを使いたい場合は `--sheet "シート名"` を付けてください。

### 3. ブラウザで開き直す

`index.html` を開き直す（開いていれば再読み込み）だけで、新しいデータが反映されます。

## 注意事項

- 「コンバージョン」列には scroll_depth などのマイクロCVが含まれる場合があり、
  CVアクション内訳の合計と一致しないことがあります（画面にも注記あり）。
- 期間の記載（例「2026年5月20日 - 2026年7月9日」）がファイル先頭にあると、
  曜日ごとの日数を自動計算して「1日あたり平均」補正が使えます。
  見つからない場合、補正チェックボックスは非表示になります（他の機能は動きます）。
- `make_data.py` の実行には Python と openpyxl が必要です（CSVならopenpyxl不要）。
  入っていない場合: `pip install openpyxl`

## 別フォルダに複製して使う

このフォルダごとコピーすれば、別案件・別期間のダッシュボードを並行して持てます。
コピー先で `make_data.py` を実行してデータだけ差し替えてください。
